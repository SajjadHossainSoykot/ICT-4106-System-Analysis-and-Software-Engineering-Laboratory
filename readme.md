# 🧩 System Analysis and Software Engineering Laboratory

This repository contains lab experiments and system design work developed as part of the course:

**ICT-4106 – System Analysis and Software Engineering Laboratory**

---

## 📘 Course Overview

This laboratory course focuses on analyzing real-world problems and transforming them into structured software system designs using standard software engineering methodologies.

Instead of direct coding, the course emphasizes **system thinking, modeling, and design**, helping students understand how complex systems are planned before development.

The designs are created using professional diagramming tools like **draw.io** and UML-based modeling.

---

## 🎯 Learning Objectives

- Understand real-world problem analysis  
- Design structured software systems  
- Learn Software Engineering methodologies  
- Model systems using UML diagrams  
- Analyze relationships between system components  
- Prepare professional lab reports  

> The goal is to think like a **Software Architect**, not just a programmer  

---

## 🧪 Laboratory Work

Each experiment focuses on designing a complete system based on real-world scenarios.

### Systems Covered

- 🏥 Hospital Management System (HOMS)  
- 📚 Library Management System (LIMS)  
- 🎓 Student Information System (SIS)

---

## 🧱 System Design Approach

Each project follows a structured **4-step methodology**:

### 1️⃣ Requirements Engineering
- Functional Requirements (What the system does)  
- Non-Functional Requirements (Performance, security, usability)

### 2️⃣ Use Case Modeling
- Identify **Actors**  
- Define **Use Cases**  
- Show interactions between users and system  

### 3️⃣ Structural Design (Class Diagram)
- Define system classes  
- Attributes and methods  
- Relationships (Inheritance, Association)

### 4️⃣ Database Design (ER Diagram)
- Identify entities  
- Define relationships  
- Use Primary Keys & Foreign Keys
